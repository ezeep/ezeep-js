üîê Pre-Authentication Example

This example shows how to pre-authenticate users with an existing refresh token, bypassing the OAuth login flow to provide a seamless printing experience.

üîí Security Notice: Refresh tokens are highly sensitive data and should NEVER be hard-coded in client code. In a production environment, you should:
  • Store tokens securely on your backend
  • Retrieve tokens via secure API endpoints
  • Use HTTPS for all communication
  • Implement token rotation

How it Works

  • An existing refresh token is used to obtain an access token
  • The setAuthRefreshToken() method handles token exchange automatically
  • Users don't need to log in again via OAuth
  • Print dialog can be opened immediately

Workflow Diagram

1
User authenticates initially via your backend system
2
Backend receives and stores refresh token securely
3
Frontend loads ezeep.js component
4
Frontend retrieves refresh token from backend
5
setAuthRefreshToken() is called
6
ezeep.js exchanges refresh token for access token
7
User is authenticated - print dialog can be opened

üí° Use Cases

  • Single Sign-On (SSO): Integration with existing authentication systems
  • Enterprise Solutions: Centralized user management via backend
  • Mobile Apps: Seamless print integration without repeated logins
  • Kiosk Systems: Pre-authenticated sessions for quick access

üîß Backend Integration Code Example

// Frontend - Retrieve token from backend
async function getRefreshToken() {
  const response = await fetch('/api/auth/refresh-token', {
    credentials: 'include' // Session cookies
  })
  const data = await response.json()
  return data.refreshToken
}

// Perform pre-authentication
customElements.whenDefined('ezp-printing').then(async () => {
  const ezpPrinting = document.querySelector('ezp-printing')
  const refreshToken = await getRefreshToken()

  await ezpPrinting.setAuthRefreshToken(refreshToken)
  console.log('User pre-authenticated successfully')
})

üìù Usage Notes

  • Open this file via an HTTPS server
  • The refresh token must come from a previous successful authentication
  • Tokens should have an expiration date and be renewed regularly
  • With invalid tokens, the system falls back to the normal OAuth flow
