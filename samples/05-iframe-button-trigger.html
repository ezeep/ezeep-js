This example demonstrates the iframe embedding method with a button trigger. Third-party apps can integrate printing without OAuth configuration!

üì¶ iframe Embed Method: This example uses the new iframe embedding approach where:
  • No Client ID or Redirect URI needed in your code
  • Authentication is handled in the hosted service
  • No CORS issues - communication via postMessage
  • Simple integration - just include one script

How it Works

  • Include the ezeep-embed.js library
  • Create an EzeepEmbed instance with trigger="button"
  • The print button appears in a modal when you call print()
  • File is provided via a URL (fileurl)
  • All authentication happens in the hosted service iframe

Configuration Used

  • trigger: "button": Shows ezeep print button in iframe
  • theme: "violet": UI color theme
  • appearance: "system": Matches system light/dark mode
  • No clientId required - configured in hosted service!

üìù Usage Notes

  • This can run on any domain - no OAuth registration needed
  • First time users will see an authentication dialog in the iframe
  • Authentication is saved for future use
  • Print dialog appears in a modal overlay

üí° Benefits vs Direct Integration

  • Simpler: No Client ID management
  • Portable: Works from any domain
  • Secure: OAuth handled in isolated iframe
  • Maintained: Updates happen automatically
