üîê Pre-Authentication Example (iframe Embed)

This example shows how to pre-authenticate users with a refresh token using the iframe embedding method. Perfect for SSO integration and enterprise solutions!

üì¶ iframe Embed Method: Pre-authentication with iframe embedding:
  • Call setAuthRefreshToken() to bypass OAuth login
  • Token is sent securely to iframe via postMessage
  • Works from any domain without configuration
  • Perfect for SSO and backend-managed auth
üîí Security Notice: Refresh tokens are highly sensitive! In production:
  • Store tokens securely on your backend
  • Retrieve tokens via secure API endpoints
  • Use HTTPS for all communication
  • Implement token rotation and expiration
  • Never hard-code tokens in client code

How it Works

  • Your backend manages user authentication
  • Frontend retrieves refresh token from backend API
  • Call printer.setAuthRefreshToken(token)
  • Token is transmitted securely to iframe
  • ezeep exchanges refresh token for access token
  • User is authenticated - no login dialog needed!

üîß Backend Integration Example

// Frontend - Retrieve token from backend
async function getRefreshToken() {
  const response = await fetch('/api/auth/ezeep-token', {
    credentials: 'include'  // Include session cookies
  });
  const data = await response.json();
  return data.refreshToken;
}

// Initialize printer with pre-authentication
const printer = new EzeepEmbed({
  hostUrl: 'http://app.ezeep.com/ezprinting'
});

// Pre-authenticate when ready
printer.on('ready', async () => {
  try {
    const token = await getRefreshToken();
    await printer.setAuthRefreshToken(token);
    console.log('‚úÖ User pre-authenticated!');
    // Enable print functionality
    enablePrintButton();
  } catch (error) {
    console.error('‚ùå Pre-auth failed:', error);
    // Fallback to normal OAuth flow
  }
});

üìù Usage Notes

  • Works from any domain - no special configuration
  • Token must be valid and not expired
  • Invalid tokens fall back to normal OAuth flow
  • Implement token refresh in your backend
